Node.js

Node is a JavaScript runtime environment that executes JS code outside of a browser. It makes it possible to develop both front- and backend in JS (fullstack). It is built using an event-driven architecture capable of asynchronous I/O. These design choices aim to optimize throughput and scalability in web applications with many input/output operations, as well as for real-time Web applications such as chat, streaming, online games etc.

To run a Node.js server, we use the command node in the command-line followed by the main .js file (usually called app.js, main.js or index.js).

NPM is the package manager in Node.js. Packages can be downloaded from the public NPM website, which can be used within a Node.js server.

To initialse a Node.js project we use the command npm init. This will create the file package.json, which includes metadata about our project, scripts, dependencied and more.

Packages can be installed using the command npm install <name_of_package>.

// package.json {
    "dependencies": {
        "express": "^4.17.1"   // express package installed with npm install --save
    },
    "devDependencies": {
        "cross-env": "^7.0.3"  // dev dependency cross-env installed with npm install --save-dev
    },
    "scripts": {                  // scripts that can be used with npm run
        "start": "node app.js",
        "start-dev": "cross-env PORT=3000 nodemon app.js"
    }
}

Node packages can be downloaded from the npm website.

Some example packages used in this server:

  • Express.js: A web framework for Node (described in further detail on the express page).
  • Nodemon: a tool that helps develop node.js based applications by automatically restarting the node application when file changes in the directory are detected.
    Usage: nodemon <main_js_file>
  • cross-env: Run scripts that set and use environment variables across platforms.
    Usage in package.json 
    "scripts": {
    "start": "node app.js",
    "start-dev": "cross-env PORT=3000 nodemon app.js"
}
// two scripts are defined: "start" and "dev-start". 
// The first will run with node command
// the second will use cross-env to set port and use nodemon commmand
// these scripts are used with the command npm run <script>
